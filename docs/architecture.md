# Architecture ‚Äî per-module reference

The exhaustive module-by-module inventory of the repository. The [README](../README.md#layout) carries the top-level directory map (what each directory *is*); this file is the detailed catalogue (what each file *does*). Keep the two in sync when a module is added, split, or removed ‚Äî the README map is a dozen lines, this is the full list.

## `src/` ‚Äî non-UI Python

- `melcloud_client.py` ‚Äî async auth + fetch + control (the shared core).
- `list_devices.py` ‚Äî CLI that prints each unit's live state.
- `sma_client.py` ‚Äî async read of the local SMA solar/energy devices (meter + inverter).
- `list_energy.py` ‚Äî CLI that prints the live energy flow.
- `energy_history.py` ‚Äî SQLite store + rollups for the energy dashboard history. Connection mechanics come from `_sqlite.py`.
- `telemetry.py` ‚Äî unified event + reading store (SQLite; #283/#289). Narrow `readings`/`events` tables with JSON sidecars; producers (alarm/power/presence/plug/RISCO) mirror their events in, the Home **Activity log** reads them back. Connection mechanics come from `_sqlite.py`.
- `hvac_automation.py` ‚Äî UI-free persistence + control law for per-unit dynamic temperature rules and daily schedules.
- `tariff.py` ‚Äî electricity tariff model: prices grid energy per time-of-use period and values self-consumed PV (the cost & savings breakdown). UI-free, graceful flat-rate default.
- `tuya_client.py` ‚Äî Smart Life / Tuya discovery and local LAN control foundation.
- `ups_client.py` ‚Äî local USB UPS status reader: prefers NUT (`upsc` against the portable Windows install), falls back to a one-shot NUT USB-HID probe, then Windows `Win32_Battery`.
- `host_shutdown.py` ‚Äî thin wrapper over Windows `shutdown.exe`: `initiate_shutdown()` schedules a graceful shutdown with a grace window, `cancel_shutdown()` aborts a pending one. Hard-blocked under pytest so tests can never trigger a real shutdown. Used by the UPS low-battery safety trigger (see `power_monitor.py`).
- `hyperv_client.py` ‚Äî status + lifecycle control for the Home Assistant Hyper-V VM (#240): shells out to `Get-VM` / `Start-VM` / `Stop-VM` for the single VM named by `HA_VM_NAME` (by name only ‚Äî never any other VM), returning a flattened `HyperVState` (state, uptime, IP, MAC). Distinct errors for missing name / not-found / insufficient rights / already-in-state; graceful ACPI stop only.
- `ha_client.py` ‚Äî UI-free async HA REST/WebSocket client (#239): discovers Assist satellites through HA-owned entity/device/area registries, joins companion media-player volume, announces to one satellite, and reads the bounded Assist pipeline debug surface; includes the pure raw-trace ‚Üí compact-interaction normalizer.
- `voice_client.py` ‚Äî async Voice Transcriber session-API client (#239): attributed create, one-second raw chunk append, SSE pass-through, finish, and single-shot fallback. It never owns audio/model persistence; Voice Transcriber does.
- `elgato_client.py` ‚Äî Elgato lights discovery/read/control over the local LAN HTTP API.
- `_atomic_json.py` ‚Äî shared atomic write-tmp + `os.replace` primitives (`atomic_write_bytes` / `write_json_atomic`; issue #327), the single implementation behind the pattern that used to be copy-pasted across 18 `src/` modules.
- `_sqlite.py` ‚Äî shared WAL-mode SQLite `connect()` context manager (issue #452), the single implementation behind the pragma/`mkdir`/`Row`-factory shape that used to be copy-pasted byte-for-byte across `energy_history.py`, `network_history.py`, and `telemetry.py`.
- `_schedule_store.py` ‚Äî shared read/save + `safe_id`/`clean_time`/`clean_days` helpers for weekly schedule-entry stores (issue #327), deduping `security_schedules.py` and `wake_alarms.py` (`hvac_automation.py` reuses the read/save pair too); built on `_atomic_json.py`.
- `_toggle_prefs.py` ‚Äî parametrized load/save for frozen bool-toggle-dataclass prefs stores (issue #327), deduping `alarm_notify_prefs.py` and `power_notify_prefs.py`; built on `_atomic_json.py`.
- `risco_client.py` ‚Äî async RISCO Cloud alarm state (incl. system AC-power + per-zone trouble flags), controls, event log, and detector bypass. Merges the pyrisco Cloud status with `risco_webui.py`'s scraped state flags.
- `risco_webui.py` ‚Äî the native RISCO WebUI HTML/JSON scraper (site login, arm/disarm command, raw state-flags read) that `risco_client.py` layers on top of ‚Äî split out in #328 to keep this brittle, screen-scraped path (the source of the false-intrusion bug fixed in #307) visually separate from the typed cloud fetcher.
- `security_schedules.py` ‚Äî UI-free persistence and due-window checks for weekly alarm schedules; read/save + field-cleaning via `_schedule_store.py`.
- `wake_alarms.py` ‚Äî UI-free persistence + due-window checks for **wake alarms** (recurring day-of-week or one-shot date; Alexa-style, ring/notify only ‚Äî never touches the RISCO security alarm), plus the voice helpers `parse_spoken_alarm` (spoken time/schedule ‚Üí entry) and `describe_alarm` (speakable summary) ‚Äî both bilingual via a `lang="es"` branch that parses/speaks Spanish times ("las siete y media entre semana", #466) ‚Äî and `next_fire`/`soonest_enabled`, used by the voice API (#306). Shares `security_schedules.py`'s read/save + field-cleaning via `_schedule_store.py`; `config/wake_alarms.json`.
- `wake_timers.py` ‚Äî in-memory (unpersisted) countdown-**timer** store; mirrors Home Assistant's own ephemeral voice timers (a restart clears them). Create/list/cancel + `mark_expired`.
- `voice_commands.py` ‚Äî the curated catalogue of every wired Voice PE command (#437), grouped by feature and tagged per language/wake word ("Okay Nabu" English, "Hey Mycroft" Spanish; the alarm + wake-alarm groups are bilingual since #466, which also added the card's EN/ES filter toggle), served to the PWA's "What can I say?" card by `routers/voice_commands.py`. Static prose, no I/O and no secrets (the disarm code is only ever the `<your code>` / `<tu c√≥digo>` placeholder `docs/voice-pe-config/README.md` already publishes). Curated rather than derived from `custom_sentences/*.yaml` because those are hassil templates and don't cover the multi-turn grocery automation or the HA built-ins; the cost ‚Äî a second place to update ‚Äî is paid by step 6 of `docs/voice-commands-howto.md`'s recipe.
- `presence_client.py` ‚Äî read-only iCloud Find My spike client for location/presence feasibility.
- `network_client.py` ‚Äî async home-network orchestrator (issue #197 split): imports the three sub-modules below and exposes the unchanged public surface (`fetch_network_state`, `resolve_ip_by_mac`, dataclasses) so callers don't move.
- `network_types.py` ‚Äî shared dataclasses, exceptions, and leaf helpers used across all network sub-modules.
- `network_ap.py` ‚Äî NETGEAR R9000 AP: device inventory (MAC/IP/name/signal/band/SSID), AP health, `reboot_access_point`, MAC rediscovery.
- `network_router.py` ‚Äî Vodafone ZXHN F6600P (ZTE): SHA256-challenge login, WAN/DHCP reads, binding write-back, `reboot_router`.
- `network_host.py` ‚Äî host-side internet probes: ping latency + packet loss, optional speedtest, `netsh wlan`.
- `list_network.py` ‚Äî CLI that prints the live network state and inventory.
- `dhcp_plan.py` ‚Äî UI-free, network-free DHCP reservation planner (#170): classifies each device into a category range from `config/dhcp_plan.json` and assigns the lowest free IP, anchoring stability on the router's existing bindings and tagging each row reserved/create/change; warns on overflow/overlap/unclassified/randomised-MAC.
- `list_dhcp_plan.py` ‚Äî CLI that prints the categorised reservation plan (mirrors `list_network.py`); `--apply` pushes the create/change rows to the router (#176) behind a `yes` prompt.
- `network_display_names.py` / `network_wifi_display_names.py` / `network_hidden.py` ‚Äî Network-tab label and hidden-state stores for attached devices and Wi-Fi radios; reuse `display_names.py` atomic load/save/set verbatim, parallel to the unit/plug/detector stores. `network_hidden.py` covers both attached-device (`config/network_hidden.json`) and Wi-Fi radio (`config/network_wifi_hidden.json`) hidden state in one module.
- `network_oui.py` ‚Äî offline device identification: bundled trimmed OUI‚Üívendor table, randomised-MAC detection, and a category/icon heuristic (no network call, render-time).
- `network_history.py` ‚Äî per-MAC history store (SQLite, modeled on `energy_history.py`; issue #129 Phase 4): first/last/times-seen, the `important` flag, and the online/offline + new-device derivations. Recorded on each `/api/network` read (no background sampler ‚Äî the AP read is expensive and tab-gated); randomised MACs are never tracked. Kept separate from the rename/hidden stores; gitignored `webapp/network_history.sqlite3`. Connection mechanics come from `_sqlite.py`.
- `dhcp_overrides.py` ‚Äî per-MAC category overrides chosen in the Network tab UI; persisted to gitignored `config/dhcp_overrides.json` and merged over `config/dhcp_plan.json` by the planner so a UI choice wins without editing the committed config.
- `camera_client.py` ‚Äî async, UI-free camera core (issue #161): ONVIF for discovery/profiles/PTZ (`ContinuousMove` / `Stop` / `AbsoluteMove`, preset management), RTSP + ffmpeg for snapshot / MJPEG stream / clip recording. Returns a flattened `CameraInfo` per declared camera; capabilities (`ptz_presets`, `ptz_absolute`, `ptz_relative`) gate the UI controls. Cameras declared in gitignored `config/cameras.json`.
- `camera_ffmpeg.py` ‚Äî ffmpeg subprocess wrapper used by `camera_client.py` for RTSP‚ÜíJPEG snapshot, RTSP‚ÜíMJPEG stream transcoding, and server-side mp4 recording. Every ffmpeg call is dispatched to a lazily-started background thread running its own persistent `ProactorEventLoop` (issue #399) ‚Äî Windows only supports `asyncio` subprocess transports on the proactor loop, but the webapp's own uvicorn loop is the selector loop (issue #397's fix for the accept-loop wedge), so ffmpeg calls can't run directly on the server's own loop.
- `camera_display_names.py` ‚Äî custom label store for camera IDs; parallel to `display_names.py`; persisted to gitignored `config/camera_display_names.json`.
- `camera_presets.py` ‚Äî software PTZ preset store for cameras that lack native ONVIF preset support; persisted to gitignored `config/camera_presets.json` (absolute pan/tilt/zoom coordinates).
- `camera_preset_names.py` ‚Äî custom label store for camera preset tokens, keyed by `"camera_id::token"`; persisted to gitignored `config/camera_preset_names.json` via `display_names.py`'s atomic load/save (issue #452), only the composite key and on-disk path differ.
- `pv_forecast.py` ‚Äî Open-Meteo global-tilted-irradiance ‚Üí expected-generation forecast engine (issue #39): one keyless API call per request for yesterday/today/tomorrow; scales hourly GTI by kWp √ó performance_ratio. UI-free; read by `GET /api/energy/forecast`.
- `pv_system_config.py` ‚Äî PV array config loader (`config/pv_system.json`): kWp, tilt, azimuth, performance ratio. Missing file returns a disabled sentinel rather than an error.
- `location_config.py` ‚Äî home-location config loader (`config/location.json`): lat/lon/optional label used by the weather and PV-forecast endpoints. Missing file surfaces as `configured=False` with HTTP 200 ‚Äî never a 500.
- `elgato_display_names.py` ‚Äî custom label store for Elgato lights; keyed by MAC address when available (survives DHCP changes), with a legacy host:port fallback; persisted to gitignored `config/elgato_display_names.json`.
- `tuya_hidden.py` ‚Äî hidden-state store for Tuya plugs/blinds; persisted to gitignored `config/tuya_hidden.json`; parallel to `network_hidden.py` and `security_hidden.py`.
- `security_hidden.py` ‚Äî hidden-state store for RISCO detectors; persisted to gitignored `config/security_hidden.json`.
- `security_trouble_ignore.py` ‚Äî per-zone trouble-ignore store; persisted to gitignored `config/security_trouble_ignore.json`; a muted zone is dropped from the main-card trouble count while still showing `Trouble ‚Äî ignored` in the list.
- `security_override.py` ‚Äî per-zone "bypass after N repeated alarms this armed session" rule store (issue #341, `max_retries` clamped to 1-3); persisted to gitignored `config/security_override.json`.
- `security_override_session.py` ‚Äî persisted scan cursor + per-zone trigger counts + auto-bypassed-zone tracking for `security_override_automation.py`, so a restart mid-session doesn't lose count; persisted to gitignored `config/security_override_session.json`.
- `presence_display_names.py` ‚Äî custom label store for presence person IDs; parallel to `display_names.py`; persisted to gitignored `config/presence_display_names.json`.
- `presence_hidden.py` ‚Äî hidden-state store for Find My / presence entities; persisted to gitignored `config/presence_hidden.json`.
- `presence_engine.py` ‚Äî webhook-backed presence state and alarm-transition decision engine: reads `config/presence_state.json`, evaluates grace-period / Kids-home logic, and drives arm/disarm; appends a JSONL audit row to gitignored `logs/presence_triggers.jsonl`.
- `presence_places.py` ‚Äî the "where's mom/dad" voice locator's named-place store (issue #438): `PresencePlace` dataclass (`id`/`label`/`lat`/`lon`/`radius_m`), whole-list CRUD (modeled on `security_schedules.py`) persisted to gitignored `config/presence_places.json`, and `resolve_place()` ‚Äî the closest configured place within its radius wins, else "Home"/"Away"/unknown from cached distance-from-home data (no new iCloud locate cost).
- `presence_roles.py` ‚Äî household-role alias store (e.g. "dad"/"mom" ‚Üí a person/entity id) for the same locator; reuses `display_names.py`'s atomic store verbatim (like `presence_display_names.py`), persisted to gitignored `config/presence_roles.json`; `resolve_person()` matches a spoken name/role case-insensitively against roles, display names, then raw names.
- `push_notifications.py` ‚Äî best-effort Web Push sender for presence transitions: reads VAPID keys + subscriptions from gitignored `config/push_config.json` / `config/push_subscriptions.json`; silent no-op when not configured.
- `notify/` ‚Äî the universal Telegram notifier vendored verbatim from `project-scaffolding` (`TelegramNotifier` + `NotifierError` + `TelegramConfig` + `build_notifier`); `notify_config.py` wires the app's gitignored `config/notify_config.json` / `TELEGRAM_*` env to it (`build_alarm_notifier()` ‚Üí notifier or silent no-op).
- `activity_log.py` ‚Äî reusable append-only JSONL activity log; `append_activity(consumer, event)` writes one timestamped line to gitignored `logs/<consumer>.jsonl`. Used for `logs/alarm.jsonl` (every arm/disarm command + result) and delegated to by the presence trigger log.
- `alarm_notify_prefs.py` ‚Äî seven per-event toggles for alarm Telegram alerts (arm/disarm successes default off; `error`/`intrusion`/`ac_lost` default on), persisted to gitignored `config/alarm_notify_prefs.json` via `_toggle_prefs.py`.
- `power_notify_prefs.py` ‚Äî three toggles (`power_lost` / `power_restored` / `auto_shutdown_low_battery`, all default on) for UPS mains-power Telegram alerts and the low-battery safety auto-shutdown, persisted to gitignored `config/power_notify_prefs.json` via `_toggle_prefs.py`.
- `list_security.py` ‚Äî CLI that prints the live RISCO alarm state (mirrors `list_devices.py`).
- `list_elgato_lights.py` ‚Äî CLI that prints + controls Elgato lights; supports `--id`, `--on`, `--brightness`, `--kelvin`.
- `list_cameras.py` ‚Äî CLI that prints camera reachability and last-snapshot metadata.
- `list_presence.py` ‚Äî CLI that prints visible iCloud Find My presence entities; accepts `--2fa-code` for fresh session auth.
- `webapp_config.py` ‚Äî webapp host/port + auth secrets loader.
- `display_names.py` ‚Äî maps unit IDs ‚Üí custom display-name overrides; persisted atomically to `config/display_names.json` (gitignored, `config/display_names.sample.json` committed). `tuya_display_names.py` (plugs, `config/tuya_display_names.json`) and `security_display_names.py` (RISCO detectors, keyed by zone id, `config/security_display_names.json`) reuse this module's atomic load/save/set verbatim ‚Äî only the on-disk path differs ‚Äî so the three rename stores never diverge.
- `static_versioning.py` ‚Äî build identity (git SHA) + content-hash (`?v=`) stamping of the PWA's `.js`/`.css` URLs so a mobile PWA never serves stale cached code.
- `alarm_scene.py` ‚Äî alarm-triggered scene capture + vision-LLM verdict core (issue #162), UI-free: `capture_scene` drives each detector-paired camera to its PTZ preset and grabs a fresh JPEG; `analyze_scene` sends it (plus the camera's last calm baseline) to a vision-capable model via the local hub for a real-vs-false-alarm verdict. Any hub/parse failure degrades to a "verify manually" verdict rather than propagating into the alarm path.
- `alarm_scene_config.py` ‚Äî persisted detector‚Üícamera+preset pairings for alarm scene capture (issue #162); only cameras paired to the detector that fired are snapshotted. Gitignored `config/alarm_scene_pairings.json`, atomic load/save shape shared with `security_schedules.py` / `display_names.py`.
- `alarm_scene_cursor.py` ‚Äî persisted scan cursor for the alarm-scene event-log read (issue #325): diffs RISCO's event log against the last-processed alarm-event timestamp so a second real alarm within one still-armed session (or a restart mid-session) still gets its own capture, instead of relying on the system-wide latch flags.
- `telemetry_adapters.py` ‚Äî per-domain reading adapters (#290) that map each domain's live reading object(s) into the flat `(entity, metric, value)` rows `src/telemetry.py`'s store records; pure and UI-free, so they unit-test directly against the device clients' own dataclasses.
- `_mac.py` ‚Äî single home for MAC-address key normalisation (upper-case, separators as reported, whitespace-trimmed), deduping what used to be copy-pasted `normalize_mac` helpers in `dhcp_plan` / `network_oui` / `network_display_names` / `dhcp_overrides`.
- `camera_token.py` ‚Äî short-lived HMAC-signed scoped tokens (60 s TTL, stateless) for camera stream/snapshot URLs, so the long-lived bearer token never has to appear in an `<img src>` URL.

## `app/webapp/` ‚Äî the FastAPI + PWA product

- `server.py` ‚Äî `create_app()`, middleware, caching static mount, routers, background-task lifespan.
- `middleware.py` ‚Äî bearer-token / loopback auth gate.
- `manager.py` ‚Äî adopt-or-spawn / restart / stop for the uvicorn webapp (used by the tray).
- `sampler.py` ‚Äî background energy sampler owned by the webapp lifecycle.
- `automation.py` ‚Äî background HVAC automation evaluator (dynamic setpoint rules + schedules) owned by the webapp lifecycle.
- `security_automation.py` ‚Äî background weekly alarm-schedule evaluator owned by the webapp lifecycle.
- `wake_alarm_automation.py` ‚Äî background wake-alarm + timer evaluator owned by the webapp lifecycle: fires due alarms (marks "ringing", best-effort Telegram notify, rearms weekly / auto-disables one-shot) and expires due timers.
- `presence_automation.py` ‚Äî background presence ‚Üí alarm automation consumer (evaluates webhook-backed presence state and fires arm/disarm) owned by the webapp lifecycle.
- `security_override_automation.py` ‚Äî auto-bypasses a detector after repeated same-session alarms per its configured `security_override.py` rule (issue #341); reuses the event-log-diff architecture issue #325 introduced (diffs `fetch_events()` against a persisted cursor rather than the live intrusion flags) and rides `presence_automation.py`'s single RISCO read instead of a second poller. Restores every auto-bypassed zone on the next arm event; logs via `src.telemetry.record_event` (`auto_bypass` / `auto_unbypass`), surfaced by `GET /api/activity`.
- `alarm_notify.py` ‚Äî single `record_alarm_action()` entry point used by the schedule, presence, and manual arm/disarm paths: always appends to the `logs/alarm.jsonl` activity log, and sends a Telegram alert only for automatic sources whose toggle is on (manual never notifies; errors de-dupe to once/day). Also `record_security_event()` / `check_security_transitions()` for edge-triggered intrusion + panel-AC alerts off the presence loop's RISCO read.
- `power_monitor.py` ‚Äî background task (webapp lifespan) that polls the UPS every `POWER_MONITOR_POLL_INTERVAL_S` in a worker thread, fires `power_notify.record_power_event()` on each mains‚Üîbattery transition (the server-side watcher behind the power-loss Telegram alerts), and enforces the low-battery safety shutdown: edge-triggers `power_notify.record_low_battery_shutdown()` once per outage when on-battery runtime drops to 15 min or below (fires even on the monitor's first observation, unlike the mains-transition alerts), cancelling via `record_low_battery_shutdown_cancelled()` if mains power returns first.
- `power_notify.py` ‚Äî `record_power_event()`: appends UPS power transitions to `logs/power.jsonl` and sends a Telegram alert per the `power_lost`/`power_restored` toggle. `record_low_battery_shutdown()` / `record_low_battery_shutdown_cancelled()`: the safety-net pair ‚Äî always logs, and when `auto_shutdown_low_battery` is on, sends a distinct Telegram alert and calls `src.host_shutdown` to schedule/cancel the OS shutdown.
- `presence_refresher.py` ‚Äî bounded background iCloud Find My diagnostic refresher; browser polling reads its in-memory cache via `GET /api/presence`, so the expensive Apple call happens only here.
- `alarm_scene_automation.py` ‚Äî alarm-triggered scene capture + AI verdict orchestrator (issue #162): the glue between the RISCO alarm and `src.alarm_scene`. Does not poll RISCO itself ‚Äî rides the presence loop's single `SecurityState` read each tick, does the cheap due-check inline, and fires the expensive event-log-read + capture + vision-call work as a detached task. Onset detection reads RISCO's event log rather than the live per-zone/system flags (issue #325), via `src.alarm_scene_cursor`.
- `telemetry_sampler.py` ‚Äî background telemetry-reading sampler owned by the webapp lifecycle; a sibling of `sampler.py` (which owns the SMA energy series) that snapshots the other device domains (HVAC temps, plug watts, UPS load, Elgato lights) into `src.telemetry`'s `readings` table on a gentle default 5-minute cadence, gated per-domain plus a master `TELEMETRY_SAMPLER_ENABLED` switch.
- `ha_trace_collector.py` ‚Äî best-effort Assist debug ingestion (#239): 15-second loop, fetches only unseen completed runs, writes compact `ha_voice` telemetry events, and caps in-memory dedupe at 256 IDs. No raw traces/audio retained.
- `_env.py` ‚Äî shared `.env` knob parsers (`_env_bool` / `_env_int`) for the webapp's background-task modules (`presence_automation` / `presence_refresher` / `security_automation` / `automation` / `telemetry_sampler` / `routers.presence`), so the "blank ‚Üí default, invalid ‚Üí warn-and-default" behaviour can't drift between the copy-pasted versions each loop used to carry.
- `event_loop.py` ‚Äî Windows event-loop shim for uvicorn (issue #396): a selector-loop factory wired into every place that spawns `app.webapp.server:app`, fixing a proactor-loop wedge where an aborted client connection (WinError 64) closed the listening socket and left `:8447` unresponsive until a tray restart.
- `_zone_lookup.py` ‚Äî shared RISCO zone id ‚Üí display name lookup (issue #401), following the `_env.py` cross-module-helper precedent; centralizes the `_zone_name_for` helper `alarm_scene_automation.py` and `security_override_automation.py` used to carry byte-identical copies of.
- `_task_loop.py` ‚Äî shared `run_loop(tick_fn, interval_s, ...)` background-loop shell (issue #452): the `while True: try/except-log/sleep` wrapped in an outer `try/except asyncio.CancelledError` that `automation.py` / `presence_automation.py` / `presence_refresher.py` / `power_monitor.py` / `security_automation.py` / `wake_alarm_automation.py` / `sampler.py` / `telemetry_sampler.py` used to hand-roll byte-for-byte.
